LampShop — микросервисы

Учебный пример на порту 8000 с путём /items — это другой проект. Здесь каталог и заказы разнесены по ТЗ:

Product Service (порт 3001)
  • Swagger UI — /docs
  • ReDoc
Order Service (порт 3002)
  • Swagger UI — /docs (сначала Authorize → X-Session-Id)
  • ReDoc
  • Покупатель: POST /api/v1/auth/register и POST /api/v1/auth/login (email + пароль). Демо-покупатель при старте (если записи ещё нет): customer@example.com / User123!.
Admin Service (порт 3003) — панель управления по ТЗ: JWT, прокси к Product/Order, журнал аудита.
  • Swagger UI — /docs
  • ReDoc
  • Вход: POST /api/v1/admin/auth/login (email + пароль). Дефолт после первого запуска: admin@lampshop.local / Admin123! (см. переменные ADMIN_BOOTSTRAP_*).
  • Фронтенд: меню профиля → «Аккаунт»: одна форма входа — сначала проверяется admin-service, затем order-service; при успехе админа открывается /admin. Прокси Vite: /admin-service → 3003, /order-service → 3002.
Быстрый старт
  1. В каждом сервисе один раз установите зависимости (из каталога сервиса): pip install -r requirements.txt — нужны пакеты PyJWT и bcrypt (импорты jwt и bcrypt).
  2. В каталоге product-service: python -m app.seed (один раз, чтобы были 20 товаров).
  3. Запустите три сервиса (см. run-dev.ps1 в корне репозитория) или вручную: 
    cd product-service
python -m uvicorn app.main:app --reload --host 127.0.0.1 --port 3001
    cd order-service
python -m uvicorn app.main:app --reload --host 127.0.0.1 --port 3002
    cd admin-service
python -m uvicorn app.main:app --reload --host 127.0.0.1 --port 3003
  4. Для админ-JWT задайте одинаковый JWT_SECRET в окружении для product-service, order-service и admin-service (или оставьте дефолт только для локальной отладки).
  5. В Order Service Swagger: Authorize → введите UUID сессии, скопируйте id товара из Product Service → POST /api/v1/cart/items → POST /api/v1/orders.

Коллекция Postman: LampShop_API_Postman.json в корне проекта.
